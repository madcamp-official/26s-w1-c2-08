# 아이템 추천 웹 서비스

## 서비스 범위

초기 MVP에서는 구매 가능한 실물 상품만 추천 대상으로 다룬다.

예시는 화장품, 전자기기, 의류, 생활용품처럼 구매 사이트에서 확인할 수 있고,
상품명, 이미지, 가격, 리뷰, 별점 등의 정보를 수집하거나 추출할 수 있는 아이템이다.

디지털 콘텐츠, 장소, 음식점, 서비스 상품처럼 구매/상품 정보 구조가 다른 대상은
초기 범위에서 제외한다.

user —- item —- review —- comment

item
- 상품 정보: 이름, 대표 이미지, 가격, 쇼핑몰/브랜드명, 원본 URL, 별점, 리뷰 수
- 반응 정보: 아이템 추천 수, 아이템 비추천 수

review
- 특정 아이템에 대해 유저가 작성한 리뷰 글
- 유저 작성 정보: 추천 이유, 리뷰 본문
- 반응 정보: 리뷰 좋아요 수, 리뷰 싫어요 수, 댓글

서비스의 핵심 컨텐츠는 아이템과 그 아이템에 달린 유저 리뷰다.
최초 유저가 어떤 상품에 대한 리뷰를 작성하면, 해당 상품이 아이템으로 DB에 추가된다.
이미 DB에 존재하는 아이템에 대해서는 다른 유저들이 별도의 리뷰를 추가로 작성할 수 있다.

기본 흐름:

```text
유저가 상품 URL과 스크린샷을 입력한다.
AI가 상품 정보를 채운다.
유저가 추천 이유와 리뷰 본문을 작성한다.
해당 상품이 DB에 없으면 새 아이템을 생성하고 첫 리뷰를 등록한다.
해당 상품이 이미 DB에 있으면 기존 아이템에 새 리뷰를 추가한다.
다른 유저들은 각 리뷰에 좋아요/싫어요를 누르고 댓글을 남긴다.
또한 아이템 자체에 추천/비추천을 남겨 꿀템 랭킹에 반영한다.
```

댓글 예시:

```text
나도 이거 써봤는데 좋다.
나는 기대보다 별로였다.
그 가격이면 다른 상품이 더 낫다.
```


초기 MVP의 꿀템 랭킹은 복잡한 개인화 알고리즘 대신 아이템 자체의 추천 수 기반 정렬로 시작한다.

기본 점수 예시:

```text
아이템 점수 = 아이템 추천 수 - 아이템 비추천 수
```

아이템 상세 화면 안에서 리뷰 목록은 리뷰 좋아요 수 기반으로 정렬한다.

```text
리뷰 점수 = 리뷰 좋아요 수 - 리뷰 싫어요 수
```

좋아요를 많이 받은 리뷰는 상단에 노출되고, 광고성 리뷰나 공감받지 못한 리뷰는 자연스럽게 아래로 내려간다.

유명 유저 추천 가중치, 팔로우한 유저 추천 가중치, 개인화 추천 알고리즘은 초기 MVP에서 제외한다.
이후 데이터가 충분히 쌓인 뒤 별도 기능으로 확장한다.

User
- 회원가입 / 로그인
  - 초기 MVP에서는 이메일 없이 username, 비밀번호, 닉네임으로 가입한다.
  - 이메일 기반 비밀번호 찾기와 알림 기능은 초기 MVP에서 제외한다.
- 다른 user를 팔로우
  - 초기 MVP에서는 제외한다.
  - 이후 개인화 추천이나 소셜 기능을 추가할 때 다시 검토한다.
- 아이템 리뷰 작성 / 아이템 등록
  - 로그인한 일반 사용자는 누구나 아이템에 대한 리뷰를 작성할 수 있다.
  - 세상에 있는 모든 아이템을 미리 등록해두지 않는다.
  - 최초 유저가 어떤 상품에 대한 리뷰를 작성하면 해당 상품이 아이템으로 DB에 추가된다.
  - 이미 DB에 존재하는 아이템에 대해서는 다른 유저가 추가 리뷰를 작성할 수 있다.
  - 초기 MVP에서는 상품 URL과 구매 사이트 스크린샷을 함께 입력받는다.
  - 스크린샷을 기반으로 AI가 상품 정보를 채워지게 한다.
    - 상품명
    - 대표 이미지
    - 가격
    - 쇼핑몰 또는 브랜드명
    - 원본 URL
    - 별점
    - 리뷰 수
  - 사용자는 AI가 채운 상품 정보에 추천 이유와 리뷰 본문을 추가해 리뷰를 작성한다.
  - URL은 상품 출처 기록과 중복 아이템 판별에 활용한다.
  - 등록 시 동일한 아이템 후보가 있으면 사용자에게 확인시킨다.
  - 중복 아이템 판별 기준
    - 원본 URL을 정규화했을 때 같으면 동일 상품 후보로 본다.
    - URL이 다르면 상품명 기준으로 기존 아이템 후보를 검색한다.
    - 상품명 후보 검색은 자동 확정이 아니라 사용자에게 "혹시 이 상품인가요?"로 보여주기 위한 용도로만 사용한다.
    - 사용자가 기존 아이템을 선택하면 해당 아이템에 리뷰를 추가하고, 아니라고 선택하면 새 아이템과 첫 리뷰를 생성한다.
    - 브랜드/쇼핑몰명 유사도 기반 자동 판별은 초기 MVP에서 제외한다.

- 아이템 삭제 요청 ADMIN
- 아이템 수정 요청 ADMIN
  - 사용자는 아이템의 상품 정보를 직접 수정하거나 삭제할 수 없다.
  - 사용자는 수정 요청 또는 삭제 요청을 보낼 수 있다.
  - 실제 수정/삭제 반영 여부는 Admin이 승인하거나 거절한다.
- 아이템 리뷰에 댓글 작성
  - 다른 유저들은 각 리뷰에 댓글을 남길 수 있다.
  - 댓글은 리뷰 글에 대한 의견, 사용 경험, 반박, 추가 정보를 담는다.
  - 한 유저는 같은 리뷰에 댓글을 여러 개 작성할 수 있다.
  - 작성자는 자신의 댓글을 수정하거나 삭제할 수 있다.
- 리뷰에 대한 좋아요 / 싫어요
  - 좋아요/싫어요의 대상은 개별 리뷰다.
  - 리뷰 좋아요/싫어요는 해당 아이템 상세 화면의 리뷰 정렬에 반영한다.
  - 한 사용자는 한 리뷰에 대해 좋아요 또는 싫어요 중 하나만 선택할 수 있다.
  - 사용자는 자신의 반응을 좋아요에서 싫어요로, 싫어요에서 좋아요로 변경할 수 있다.
  - 같은 반응을 다시 누르면 반응을 취소할 수 있다.
- 아이템 자체에 대한 추천 / 비추천
  - 추천/비추천의 대상은 아이템 자체다.
  - 아이템 추천/비추천은 이 아이템이 진짜 꿀템인지 정량적으로 나타내는 지표다.
  - 아이템 추천/비추천은 꿀템 랭킹 점수에 반영한다.
  - 한 사용자는 한 아이템에 대해 추천 또는 비추천 중 하나만 선택할 수 있다.
  - 사용자는 자신의 반응을 추천에서 비추천으로, 비추천에서 추천으로 변경할 수 있다.
  - 같은 반응을 다시 누르면 반응을 취소할 수 있다.

item
- 상품 정보
- 아이템 추천 / 비추천
- 여러 개의 리뷰

review
- 추천 이유 / 리뷰 본문
- 리뷰 좋아요 / 싫어요
- 댓글

Admin
- 수정,삭제 요청에 대한 응답
  - 초기 MVP에서는 별도 React 관리자 화면을 만들지 않고 Django admin을 사용한다.
  - Admin은 Django admin에서 아이템, 리뷰, 댓글, 수정 요청, 삭제 요청을 확인한다.
  - 수정/삭제 요청은 Django admin에서 승인 또는 거절한다.

프레임워크
Frontend - React
Backend - Django

## MVP 화면 범위

1. 회원가입 / 로그인 화면
2. 꿀템 랭킹 목록 화면
   - 아이템 자체의 추천 수 / 비추천 수 기반으로 정렬한다.
3. 아이템 상세 화면
   - 상품명
   - 대표 이미지
   - 가격
   - 쇼핑몰 또는 브랜드명
   - 원본 URL로 이동 버튼
   - 아이템 추천 수 / 비추천 수
   - 내 아이템 추천/비추천 상태와 버튼
   - 리뷰 목록
   - 리뷰 목록은 리뷰 좋아요 수 / 싫어요 수 기반으로 정렬한다.
   - 각 리뷰의 좋아요 수 / 싫어요 수
   - 각 리뷰의 댓글 목록
   - 각 리뷰의 댓글 작성 폼
   - 수정 요청 버튼
   - 삭제 요청 버튼
4. 아이템 등록 및 리뷰 작성 화면
5. 마이페이지
   - 내가 등록한 아이템
   - 내가 작성한 리뷰
   - 내가 작성한 댓글
   - 내가 보낸 수정/삭제 요청 상태
6. 수정/삭제 요청 작성 화면

Admin 화면은 React로 별도 구현하지 않고 Django admin을 사용한다.
